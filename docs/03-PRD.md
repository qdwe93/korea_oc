## 5. 디자인 시스템

```css
:root {
  --navy: #242860;        /* 주조색 — 기존 브랜드 계승 */
  --blue: #2e6db4;        /* 보조 — 로고 상단 블루 계열 */
  --sky:  #5bc2e7;        /* 보조 — 로고 하단 스카이블루 계열 */
  --orange: #ffab07;      /* 포인트 — CTA/강조에만 소량 */
  --ink: #1a1d29;  --gray: #6b7080;  --bg: #f7f8fa;  --white: #ffffff;
}
```

- 폰트: Pretendard (CDN: `cdn.jsdelivr.net/gh/orioncactus/pretendard`). 제목 700, 본문 400.
- 레이아웃: 최대 폭 1140px 컨테이너, 섹션 상하 여백 96px(모바일 56px).
- 반응형 브레이크포인트: 768px / 1024px. **모바일 퍼스트.**
- 버튼: 주조 네이비 배경/흰 글자, 포인트 CTA만 오렌지. 라운드 6px.
- 로고: SVG 텍스트 로고 리메이크 — 2단 구성 "KOREA OIL"(네이비 `--navy`) / "CHEMICAL CO.,LTD"(스카이 `--sky`) + 하단 소형 국문. 원본 참조: `assets/reference/temp_p01.jpg`.
- 금지: 캐러셀 자동 슬라이드, 진입 팝업, 과도한 스크롤 애니메이션 (신뢰·속도 저해).

## 6. 기술 요구사항

| 항목 | 요구사항 |
|---|---|
| 스택 | 순수 HTML5 + 단일 `css/style.css` + 최소 `js/main.js`(모바일 메뉴 토글 정도). 빌드 도구·프레임워크 금지 |
| 호환 | 최신 Chrome/Edge/Safari + 모바일 브라우저. IE 미지원 |
| 성능 | 페이지당 총 전송량 1MB 이하(이미지 최적화 전제), Lighthouse Performance/SEO/A11y/Best Practices 각 90+ |
| SEO | 페이지별 title·description, OG 태그, `robots.txt`, `sitemap.xml`, 구조화 데이터(JSON-LD `Organization`: 사명·주소·전화) |

| 이미지 폴백 | `assets/images/*` 부재 시에도 레이아웃이 깨지지 않도록 CSS 그라데이션 배경 폴백 |
| 배포 | FTP 업로드만으로 동작하는 상대경로 구조. 서버 사이드 코드 없음 |

## 8. 테스트 요구사항 (구현 전 작성 → 통과까지 구현)

1. **스모크 (Playwright)**: 4페이지 각각 — HTTP 200(파일 로드), 고유 `<title>` 존재, h1 존재, GNB 3링크 모두 유효 페이지로 이동, 375px 뷰포트에서 햄버거 열림/닫힘 동작.
2. **콘텐츠 가드**: 모든 페이지 푸터에 `051-816-2726`, `051-816-2728`, `koreaoil@koreaoilchem.com`, `서용환`, 주소 문자열이 정확히 존재. (연락처 오타는 최악의 결함이므로 테스트로 고정)
3. **마크업**: html-validate 통과(에러 0).
4. **링크·리소스**: 내부 href/src 무결성 — 존재하지 않는 파일 참조 0. 단, `assets/images/*`는 이미지 준비 전이므로 예외 목록으로 관리.
5. 수동 검수: Lighthouse 4항목 90+, 실기기 모바일 1회.
