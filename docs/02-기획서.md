# 02. 홈페이지 기획서 — 코리아유화주식회사

> 작성일: 2026-07-07
> 전제: `01-사전조사.md`의 조사 결과에 기반한다. 모든 결정에는 판단 근거를 함께 기록한다.

## 1. 홈페이지의 목적 정의

**"거래처·기관이 '코리아유화'를 검색했을 때, 회사를 신뢰할 수 있게 만드는 디지털 명함"**

### 판단 근거
1. 업종이 액체연료 **도매업(B2B)** 이다. 일반 소비자 대상 판매·예약·커머스 기능이 필요 없다. B2B에서 홈페이지의 실질 역할은 (a) 신규 거래 검토 시 상대방의 실사 자료, (b) 금융기관·관공서 확인용, (c) 명함/이메일 도메인의 신뢰 보강이다.
2. 현재 도메인이 15년째 "Under Construction"이다. 거래처가 검색해서 이 페이지를 보면 오히려 신뢰가 깎인다. 즉 **"없는 것보다 나쁜 상태"를 해소하는 것**이 1순위 목표다.
3. 검색 결과에 회사 정보가 전무하다. 홈페이지 하나만 제대로 올려도 "코리아유화" 검색을 독점할 수 있다.

### 목표 지표 (MVP 기준)
- "코리아유화" / "korea oil chemical" 검색 시 공식 사이트가 노출된다 (네이버·구글 등록 완료).
- 모바일·데스크톱 어디서나 3초 내 로드, 연락처까지 2클릭 이내.
- 대표가 명함·이메일 서명에 URL을 자신 있게 적을 수 있는 완성도.

## 2. 목표 형태: **4페이지 정적(Static) 기업 소개형 웹사이트**

### 사이트맵

```
/               홈        — 히어로, 회사 한줄 소개, 사업영역 요약, 연락 CTA
/about.html     회사소개   — 대표 인사말, 회사 개요, 연혁[확인필요], 오시는길 요약
/business.html  사업영역   — 취급 품목·서비스 소개 (카드형)
/contact.html   문의/오시는길 — 지도, 주소, 전화/팩스/이메일, 웹메일 링크
```

### 형태 결정의 판단 근거

**① 왜 "정적 사이트"인가 (CMS·프레임워크 배제)**

| 선택지 | 판단 | 이유 |
|---|---|---|
| 워드프레스 등 CMS | ✗ | 콘텐츠 갱신 빈도가 극히 낮은 회사다(15년간 임시 페이지 유지가 그 증거). CMS는 보안 패치·플러그인 관리가 계속 필요해서, 관리자가 없으면 해킹당한 채 방치되기 쉽다. 관리 부담 0이어야 한다. |
| React/Next.js 등 SPA/SSR | ✗ | 4페이지 소개 사이트에 빌드 파이프라인·Node 호스팅은 과잉이다. 또한 현 호스팅(PHP 웹호스팅)에 Node 배포가 불가능하다. |
| **순수 HTML/CSS/JS 정적 사이트** | ✓ | FTP 업로드만으로 **현재 호스팅에 그대로 배포 가능** (temp.php가 돌고 있는 PHP 호스팅이므로 정적 파일은 무조건 서빙됨). 한 번 올리면 썩지 않고, 유지비 추가 0원, 속도·SEO·보안 모두 유리. |

**② 왜 "멀티페이지 4장"인가 (원페이지 배제)**

- 콘텐츠 양만 보면 원페이지도 가능하다. 그러나 한국 B2B 관행상 **"회사소개 / 사업영역 / 오시는길" 메뉴가 있는 구조 자체가 신뢰 신호**다. 거래처 실무자·기관 담당자는 특정 정보(주소, 대표자, 품목)를 찾으러 오는데, 페이지가 나뉘어 있어야 찾기 쉽고 검색엔진에도 페이지별 키워드로 잡힌다.
- 4페이지를 넘기지 않는 이유: 채울 콘텐츠(연혁, 인증, 뉴스)가 아직 없다. 빈 메뉴는 오히려 회사가 작아 보이게 한다. **콘텐츠가 생기면 페이지를 늘리는 방향**이 맞다.

**③ 언어: 국문 우선, 영문은 2차**

- 영문 사명을 쓰고 무역 가능성이 있으나, 실제 수출입 여부가 미확인이다. 미확정 정보로 영문 페이지를 만들면 오역·빈약한 내용으로 역효과. **국문 완성 → 대표 확인 후 영문 추가**가 안전하다. 단, 구조는 다국어 확장이 쉽게 잡아둔다(텍스트를 콘텐츠 영역에 집중, `lang` 속성 정리).

**④ HTTPS 적용**

- 현재 자체 도메인 인증서가 없어 HTTPS 접속 시 인증서 오류가 뜬다. 크롬이 "주의 요함"을 띄우는 사이트는 신뢰 목적에 정면으로 반한다. → 호스팅사 무료 SSL(Let's Encrypt) 신청 또는 SSL 지원 호스팅으로 이전을 배포 단계에 포함한다.
- 단, **mail.koreaoilchem.com 메일이 운영 중**이므로 DNS를 건드릴 때 MX 레코드 보존을 반드시 확인한다.

**⑤ 문의 기능: 1차는 전화/이메일 직링크, 폼은 2차**

- 문의 폼은 서버 스크립트+스팸 관리가 필요하다. B2B 유류 거래는 어차피 전화로 이루어지는 업계다. `tel:` / `mailto:` 링크가 실사용에 더 맞고 관리 부담이 없다. 폼은 필요성이 확인되면 PHP 한 장으로 추가 가능하다(현 호스팅이 PHP 지원).

## 3. 디자인 방향

- **컬러**: 기존 브랜드 자산 계승 — 네이비 `#242860`(주조) + 스카이블루(로고 그라데이션 계열, 보조) + 오렌지 `#ffab07`(포인트).
  - 근거: 기존 로고·임시 페이지에서 실제로 쓰던 색이다. 로고를 다시 그리더라도 색을 유지하면 명함 등 기존 인쇄물과 충돌하지 않는다. 네이비는 에너지·해운 업계의 표준 신뢰 색상이기도 하다.
- **타이포**: Pretendard (무료, 한글 가독성 최상, CDN 제공) — 라이선스 문제 없음.
- **무드**: 항만·에너지·물류의 사진 기반 히어로 + 넉넉한 여백. "오래된 중소기업 홈페이지" 느낌(과한 배너, 팝업, 플래시 감성)을 피하는 것이 리뉴얼의 핵심 가치.
- **로고**: 기존 로고가 저해상도 비트맵뿐 → 동일 문안·유사 배색으로 텍스트 로고 리메이크(웹폰트/SVG). 원본은 `assets/reference/temp_p01.jpg` 참조.

## 4. 기술 스택 확정

| 항목 | 선택 | 근거 |
|---|---|---|
| 마크업/스타일 | HTML5 + 단일 CSS 파일(커스텀 프로퍼티) | 빌드 도구 불필요, 누구나 수정 가능 |
| 스크립트 | 바닐라 JS 최소한(모바일 메뉴, 스크롤 등) | 의존성 0 |
| 지도 | 카카오맵 퍼가기(iframe) 우선 | API 키 발급 없이 사용 가능, 국내 사용자 친화적 |
| 폰트 | Pretendard CDN | 위 참조 |
| 배포 | FTP 업로드 (기존 호스팅) → SSL 확보 | 추가 비용 0 |
| 테스트 | Playwright 스모크 + html-validate + 링크 체크 | 아래 개발 절차 참조 |
| 형상관리 | git (로컬 저장소) | 바이브 MVP 4단계의 커밋 절차 수행 |

## 5. 개발 절차 — 바이브 MVP 4단계 프로세스 적용

> 원칙: "PRD 제대로 깎아서 첫턴에 최대한 많이 → 컨텍스트 파일로 문서화 → 테스트 먼저, 통과까지 구현 → 통과 시 커밋, 검수 후 TODO 갱신 커밋"

### 1단계 — PRD 깎기 & 첫턴 최대 산출 (현재 단계 ✅)
- 사전조사(`01`) → 기획(`02`, 본 문서) → 이미지 목록(`04`) → **PRD(`03`)** 완성.
- PRD를 AI에게 투입하는 첫 턴에 **사이트 전체 스캐폴드(4페이지 + 공통 헤더/푸터 + 디자인 시스템 + 반응형)까지 한 번에** 뽑는다. 콘텐츠 미확정 부분은 `[확인필요]` 플레이스홀더로 채워 배치·디자인 검증을 먼저 끝낸다.

### 2단계 — 컨텍스트 자가 문서화
- `TODOs.md`에 작업 항목·진행 상태·결정 사항을 기록하며 진행한다. (이미 생성, 루트 참조)
- 새로 확정되는 회사 정보는 `01-사전조사.md`의 미확정 표를 갱신한다.

### 3단계 — 테스트 우선 구현
- 요구사항 단위로 **테스트를 먼저 작성**하고 통과할 때까지 구현한다. 정적 사이트의 테스트는:
  1. **Playwright 스모크**: 4페이지 각각 — 로드 성공, `<title>`·핵심 텍스트 존재, 네비게이션 링크 동작, 375px(모바일)/1280px(데스크톱) 렌더에서 메뉴 동작.
  2. **html-validate**: 마크업 유효성.
  3. **링크 체커**: 내부 링크·이미지 경로 무결성 (배포 전 깨진 링크 0 보장).
  4. **콘텐츠 가드 테스트**: 전화번호·주소·이메일이 모든 페이지 푸터에 정확히 표기되는지 문자열 검증 (오타가 가장 치명적인 정보이므로 테스트로 고정).
- 수동 검수 항목(테스트 자동화 대상 아님): Lighthouse 성능·접근성·SEO 90+ 확인, 실기기 모바일 확인.

### 4단계 — 커밋 & 검수 루프
- 테스트 통과 → 커밋 → **대표(의뢰인) 검수** → 피드백 반영 → `TODOs.md` 갱신 → 커밋.
- 검수 포인트를 명확히: ① 문안(특히 인사말·품목) ② 이미지 톤 ③ 연락처 정확성.

### 단계별 일정 감각 (참고)
| 단계 | 내용 | 예상 |
|---|---|---|
| A | PRD 투입 → 전체 스캐폴드 + 홈 완성 | 첫 턴 |
| B | 서브 3페이지 + 테스트 통과 | 1~2턴 |
| C | 이미지 생성분 반영, 콘텐츠 확정분 반영 | 검수와 병행 |
| D | 배포(FTP) + SSL + 서치콘솔/서치어드바이저 등록 | 반나절 |

## 6. 배포·오픈 체크리스트 (D단계 상세)

1. 호스팅 FTP 접속 정보 확보 (GAUTECH 관리 계정 — 대표 확인 필요)
2. 기존 `index.*`/temp.php 백업 후 새 파일 업로드
3. 자체 도메인 SSL 발급 (호스팅사 무료 SSL 또는 이전) — **MX 레코드 보존 확인**
4. `robots.txt`, `sitemap.xml` 업로드
5. 네이버 서치어드바이저 + 구글 서치콘솔 소유 확인·색인 요청
6. 카카오톡/문자로 URL 공유 시 미리보기(OG 태그) 확인
7. 구글 비즈니스 프로필 / 네이버 스마트플레이스 등록 (선택, 검색 노출 보강)

## 7. 범위 제외 (MVP에서 하지 않는 것)

- 게시판/공지사항, 관리자 페이지, 문의 폼(2차), 영문 페이지(2차), 뉴스레터, 채용 페이지
- 근거: 갱신 주체가 없는 기능은 방치되어 "마지막 글 2026년"으로 남는다. 이는 신뢰 목적에 역행한다.
